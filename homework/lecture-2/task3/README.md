# Задание 3: Function Calling через GigaChat API

## Описание

Реализация полного цикла **function calling** с использованием GigaChat API. Программа демонстрирует, как модель может самостоятельно решать вызывать ли функцию для выполнения задачи пользователя, затем формировать финальный ответ на основе результата.

## Архитектура решения

```
User Question
      ↓
[GigaChat API] ← functions + function_call: "auto"
      ↓
Модель анализирует: нужна ли функция?
      ↓
      ├─ YES: finish_reason = "function_call"
      │     ↓
      │  [Execute Function]
      │     ↓
      │  [Return Result to Model]
      │     ↓
      │  [Final Answer]
      │
      └─ NO: finish_reason = "stop"
            ↓
         [Direct Answer]
```

## Структура проекта

- **app_config.py** - Конфигурация GigaChat (API URL, модель, параметры)
- **auth.py** - Получение токена через OAuth
- **gigachat_client.py** - Клиент для работы с GigaChat API с поддержкой function calling
- **models.py** - Pydantic модели для типизации (WeatherData, FunctionCall)
- **tasks.py** - Бизнес-логика function calling, определение функций, их выполнение
- **main.py** - Точка входа, демонстрация примеров

## Использованная функция

**get_weather(city)** - Получить информацию о погоде в городе

```json
{
  "name": "get_weather",
  "description": "Получить информацию о погоде для города",
  "parameters": {
    "type": "object",
    "properties": {
      "city": {
        "type": "string",
        "description": "Название города"
      }
    },
    "required": ["city"]
  }
}
```

## Цикл обработки (run_function_calling_cycle)

1. Отправляем вопрос с доступными функциями
2. Проверяем `finish_reason` ответа
3. Если `"function_call"` - извлекаем имя и аргументы
4. Выполняем функцию (в реальности - запрос к API или БД)
5. Отправляем результат обратно модели
6. Получаем финальный ответ

## Примеры диалогов

### Пример 1: Вопрос о погоде
```
Пользователь: "Какая погода в Москве?"

1. Модель получает вопрос и доступные функции
2. Решает вызвать: get_weather(city="Москва")
3. Система выполняет функцию → результат
4. Модель получает результат
5. Модель формирует красивый ответ:
   "В Москве сейчас 5.2°C, облачно, ветер 3.5 км/ч, влажность 72%"
```

### Пример 2: Безличный вопрос
```
Пользователь: "Привет, как дела?"

1. Модель получает вопрос и доступные функции
2. Решает НЕ вызывать функцию (это не о погоде)
3. Отвечает прямо: "Привет! Я здесь, чтобы помочь с информацией о погоде."
```

## Примечания

- В данном решении используются **мок-данные** для погоды
- В реальном приложении здесь был бы запрос к API (OpenWeatherMap, Яндекс.Погода и т.д.)
- Функция полностью описана в формате GigaChat API
- Модель **самостоятельно решает** вызывать функцию, не захардкожено
